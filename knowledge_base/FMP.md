# Formation Material Physics (FMP)

*(технический документ — концепция физического движка проекта)*

---

# 1. Постановка задачи

Проект требует физической симуляции боя, в которой:

* результат столкновения определяется физикой, а не скриптами,
* поведение армии является следствием состояния бойцов,
* мораль, организованность и усталость влияют на бой физически,
* производительность позволяет симулировать сотни бойцов одновременно.

Классические подходы не решают эту задачу в полной мере:

* **Полная rigid body симуляция** — физически точна, но имеет сложность O(n²) и требует процедурной анимации для каждого бойца.
* **Raycast/shapecast подход** — производителен, но превращает игровые параметры в числовые множители, а не в физические свойства.

FMP является третьим путём.

---

# 2. Центральная идея

> Формация является физическим материалом.
> Игровые параметры армии являются механическими свойствами этого материала.

Это не метафора — это буквальная физическая модель.

| Игровой параметр | Физическое свойство материала |
|---|---|
| Организованность | Жёсткость (stiffness) |
| Мораль | Предел текучести (yield strength) |
| Усталость | Накопленная пластическая деформация |
| Тип формации | Анизотропия (разные свойства по осям) |
| Боевой опыт | Вязкоупругость (способность восстанавливаться) |

Элитная фаланга с высокой организованностью — жёсткое упругое тело:
удар отражается, строй двигается как единое целое.

Деморализованная толпа — вязкая жидкость:
давление рассеивается, бойцы текут в стороны.

Клин — анизотропный материал:
жёсткий вдоль оси атаки, мягкий с флангов.

---

# 3. Архитектура — три слоя активации

FMP разделяет симуляцию на три слоя с разной стоимостью вычислений.
Каждый слой активируется только там и тогда, где он необходим.

```
┌─────────────────────────────────────────────────────┐
│           FORMATION MATERIAL PHYSICS (FMP)           │
├─────────────────────────────────────────────────────┤
│  СЛОЙ 1: Formation Field          O(formations)      │
├─────────────────────────────────────────────────────┤
│  СЛОЙ 2: Contact Zone             O(contact_width)   │
├─────────────────────────────────────────────────────┤
│  СЛОЙ 3: Individual Physics       O(active_events)   │
└─────────────────────────────────────────────────────┘
```

Общая сложность: O(formations) + O(k) — не O(n²) и не O(n).

---

## 3.1 Слой 1 — Formation Field

**Всегда активен. Минимальная стоимость.**

Каждая формация представлена как физический материал.
Бойцы внутри формации являются точками выборки поля — не самостоятельными физическими телами.

Контакт двух формаций — это столкновение двух материалов.
Результат контакта — волна давления, распространяющаяся через строй.

Скорость и затухание волны определяются свойствами материала:

```
скорость волны   = f(организованность)
затухание волны  = f(1 − мораль)
предел текучести = f(мораль, усталость)
```

При превышении предела текучести в точке — локальный разлом.
Разлом распространяется как трещина по градиенту слабых мест.

Паника распространяется как волна через соседей:
бегущий боец снижает мораль рядом стоящих,
снижение морали снижает предел текучести,
предел текучести превышается — следующий боец бежит.

---

## 3.2 Слой 2 — Contact Zone Physics

**Активируется на границе формаций. Стоимость пропорциональна ширине контакта.**

В зоне физического контакта двух формаций —
передний ряд бойцов с каждой стороны —
активируется полная физическая симуляция.

Эти бойцы становятся настоящими физическими телами:
* с массой, скоростью, коллайдерами,
* с физическим давлением щитов и тел,
* с передачей импульса при столкновении.

Результаты Слоя 2 питают Слой 1 как граничное давление:
что происходит в переднем ряду — волной уходит вглубь строя.

Контактная зона перемещается вместе с движением формаций.
При отступлении или прорыве — зона сдвигается или расширяется.

---

## 3.3 Слой 3 — Individual Physics

**Активируется для конкретных событий. Стоимость пропорциональна количеству событий.**

Полная физика активируется для:

* **героя** — всегда, как постоянный участник Слоя 3,
* **бойца получающего удар** — на время взаимодействия,
* **бойца при моральном сломе** — переход в fluid-режим,
* **бойца вне строя** — сражающегося, бегущего, упавшего.

# 4. Geometric Impulse Computation

Удары оружием в FMP реализуются через **Geometric Impulse Computation (GIC)** —
метод, объединяющий производительность shapecast
и физическую точность impulse-based simulation.

В момент удара:

```
1. Выполняется shapecast по дуге удара
2. Из геометрии пересечения вычисляется импульс:

   impulse = weapon_mass × tip_velocity × depth_factor × angle_factor

   depth_factor — глубина пересечения shapecast с телом цели
                  (прямой удар = 1.0, касательный = 0.1..0.3)
   angle_factor — угол между вектором удара и нормалью поверхности
                  (перпендикулярный = 1.0, скользящий → 0)

3. Импульс передаётся цели как физическая сила
4. Цель временно переходит в Слой 3 для обработки реакции
```

Shapecast не является детекцией с последующим применением урона.
Shapecast является геометрическим вычислением физически корректного импульса.

Анимация является визуальным слоем отображения результата.
Анимация не определяет момент удара и не является источником хитбокса.

# 5. Два вида слома строя

FMP различает два физически разных явления:

## 5.1 Моральный слом — боец бежит

Давление превышает предел текучести (определяется моралью).
Боец покидает строй с высокой скоростью в направлении от угрозы.
Бегство заразительно: снижает мораль соседей через волновой механизм Слоя 1.

## 5.2 Физический выбой — боец выбит из строя

Смещение превышает физический порог (определяется организованностью).
Боец теряет позицию в строю, но не теряет боевой воли.
Переходит в индивидуальный режим — сражается автономно вне формации.

---

# 6. Эмерджентные тактические явления

Следующие тактические явления возникают как естественное следствие FMP
без заскриптованной логики:

**Клин эффективнее фронтальной атаки** —
потому что концентрирует давление в малой площади,
превышая локальный предел текучести точечно,
тогда как та же суммарная сила размазанная по фронту остаётся ниже порога.

**Фланговая атака разрушительна** —
потому что давление с двух сторон создаёт интерференцию волн внутри строя,
материал теряет устойчивость быстрее чем при одностороннем давлении.

**Смерть героя ломает строй** —
потому что снижение морали соседей снижает предел текучести,
то же давление которое они держали — теперь превышает порог,
разлом возникает органически в зоне потери.

**Усталость накапливается физически** —
потому что усталость увеличивает накопленную пластическую деформацию,
предел текучести снижается с каждым часом боя,
одинаковый удар в начале и конце боя — разные результаты.

# 7. Реализация в Rust/Bevy

## 7.1 Структура ECS

```
Formation (Entity)
  ├── FormationField       { stiffness, yield_strength, viscosity, anisotropy }
  ├── FormationState       { pressure_wave: Vec<f32>, fractures: Vec<usize> }
  └── FormationMembers     { soldiers: Vec<Entity> }

Soldier (Entity)
  ├── SoldierState         { morale, organization, fatigue, health }
  ├── PhysicsLayer         { layer: Layer1 | Layer2 | Layer3 }
  └── Transform, Velocity  (стандартные Bevy компоненты)

Hero (Entity)
  ├── SoldierState
  ├── PhysicsLayer(Layer3) (всегда)
  └── HeroPresence         { morale_radius, command_radius }
```

## 7.2 Порядок систем за тик

```
1. update_formation_material_properties
   (организованность/мораль/усталость → физические параметры)

2. propagate_pressure_waves
   (волны давления внутри каждой формации)

3. detect_contact_zones
   (определить где формации соприкасаются)

4. activate_contact_zone_physics
   (передний ряд → Слой 2, запуск Rapier/Avian для них)

5. process_individual_events
   (удары, сломы, бегство → Слой 3)

6. feed_results_to_formation_field
   (результаты Слоёв 2 и 3 → граничное давление в Слое 1)

7. update_soldier_positions_from_field
   (позиции бойцов Слоя 1 = интерполяция из поля)

8. propagate_panic
   (бегущие бойцы → снижение морали соседей)
```

## 7.3 Физический движок

Для Слоёв 2 и 3 рекомендуется **Avian** (bevy_avian):
* нативная интеграция с Bevy ECS,
* лучший контроль над активацией/деактивацией тел,
* поддержка kinematic + dynamic тел в одной сцене.

Слой 1 реализуется как кастомная система без внешнего физического движка.

---

# 8. Ключевые отличия от существующих подходов

FMP не является оптимизацией rigid body симуляции.
FMP является сменой модели.

В существующих подходах игровые параметры (мораль, усталость)
**влияют на результаты** физики через множители и модификаторы.

В FMP игровые параметры **являются** физическими свойствами материала.
Мораль — это буквально предел текучести.
Организованность — это буквально жёсткость.
Усталость — это буквально накопленная деформация.

Это означает что язык дизайна и язык физики совпадают.
Аксиомы проекта описывают физику движка напрямую.
