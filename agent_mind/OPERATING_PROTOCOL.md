# Операционный протокол агента

Как использовать `agent_mind` в работе над WarSim.

## Иерархия источников

1. **`knowledge_base/`** — канон (дизайн, аксиомы, FMP, устойчивые принципы).
2. **`agent_mind/`** — оперативная память (решения, гипотезы, уроки, сжатая модель).
3. Код и артефакты репозитория — фактическая реализация.

При противоречии: канон > код (если код явно устарел — зафиксировать в `DECISIONS` или `LESSONS_LEARNED`).

## Перед задачей

**Лёгкая задача** (правка текста, мелкий вопрос): достаточно `PROJECT_MODEL.md` или одного модуля.

**Серьёзная задача** (архитектура, FMP, стек, новый подсистемный дизайн):

1. [`AGENT_MIND.md`](../AGENT_MIND.md) — напоминание правил.
2. Этот README — индекс.
3. [`PROJECT_MODEL.md`](PROJECT_MODEL.md).
4. [`DECISIONS.md`](DECISIONS.md) + [`HYPOTHESES.md`](HYPOTHESES.md) по теме.
5. [`CONTEXT_MAP.md`](CONTEXT_MAP.md) → нужные файлы `knowledge_base/`.
6. При необходимости — [`modules/`](modules/).

Не читать всё подряд: только релевантное.

## После задачи

Обновлять память **только если** появилось что-то устойчивое:

| Что произошло | Действие |
|---------------|----------|
| Принято решение с последствиями | Запись в `DECISIONS.md` |
| Появилась проверяемая идея | `HYPOTHESES.md` (не в канон сразу) |
| Вопрос отложен | `OPEN_QUESTIONS.md` |
| Эксперимент / ошибка дали вывод | `LESSONS_LEARNED.md` |
| Сдвинулась общая картина | `PROJECT_MODEL.md` |
| Тема разрослась | Краткая заметка в `modules/<topic>.md` |
| Идея подтверждена и стала нормой проекта | Статус гипотезы + при необходимости правка `knowledge_base/` |
| Запись устарела | `Status: superseded` или перенос в `archive/` |

## Запреты

- Не копировать в `agent_mind` большие куски из `knowledge_base` — ссылки и сжатые выводы.
- Не писать длинные рассуждения — только то, что сэкономит время в следующей сессии.
- Не удалять историю без пометки (`superseded`, архив).

## Сжатие памяти (периодически)

После нескольких крупных итераций:

1. Сверить `PROJECT_MODEL.md` с каноном и `DECISIONS.md`.
2. Закрыть или отклонить зависшие гипотезы.
3. Перенести подтверждённое в `knowledge_base`, если ещё не там.
4. Устаревшее — в `archive/` с датой в имени файла.

Подробные форматы: [`WORKFLOWS.md`](WORKFLOWS.md).
