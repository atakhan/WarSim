# Модуль: документация проекта

Как устроены документы WarSim и как агент с ними работает.

## Два слоя

| Слой | Путь | Роль | Частота изменений |
|------|------|------|-------------------|
| Канон | `knowledge_base/` | Истина для людей и дизайна | Реже, осознанно |
| Память агента | `agent_mind/` | Решения, гипотезы, уроки, сжатая модель | Чаще |

Точка входа агента: [`AGENT_MIND.md`](../../AGENT_MIND.md).

## Файлы канона (текущий набор)

1. `PROJECT_CONSTITUTION.md` — манифест игры  
2. `PROJECT_AXIOMS.md` — аксиомы мира/армии  
3. `FMP.md` — технический движок боя  
4. `TECHNICAL_STACK_NOTES.md` — стек и лаборатория  

## Когда трогать канон

- Подтверждённая гипотеза стала принципом проекта  
- Ошибка в каноне исправлена по факту прототипа  
- Новый устойчивый термин/раздел нужен всем читателям  

Не писать в канон: сырые гипотезы, черновые рассуждения, дневник сессий.

## Когда трогать agent_mind

- Решение, гипотеза, открытый вопрос, урок — см. [`WORKFLOWS.md`](../WORKFLOWS.md)

Полная: [`CONTEXT_MAP.md`](../CONTEXT_MAP.md).
