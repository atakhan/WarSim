# Модель проекта (сжатая)

*Последнее обновление: 2026-05-16*

Операционная картина для агента. Канон: [`knowledge_base/`](../knowledge_base/).

## Что строим

**WarSim** — командная тактическая игра: игрок управляет героем как ядром **организованного отряда**, не отдельным «героем-MOBA». Бой **системный**, не визуальная имитация: исход из позиции, строя, импульса, морали, усталости, координации.

Цикл матча: фазы **Мир** (подготовка) и **Война** (бои); победа через базы. Сейчас фокус — **ядро боя**, не контент.

## Текущая фаза

**Первая Bevy/FMP-лаборатория.** Есть Rust/Bevy scaffold и runnable-заготовка Layer 1: два отряда, поле давления, локальный слом, debug UI параметров материала и анизотропия формаций через взвешенный Лапласиан. Математика Formation Field вынесена в тестируемый модуль с unit-инвариантами для профилей давления, fracture, анизотропии и morale/fatigue drift; добавлен test-only scenario harness с regression-проверками Wedge/Flank/LowMorale/OffsetContact и UI snapshot-метриками center/edge/front/rear/flank pressure/fracture. Начат Layer 2 contract: `ContactFront/ContactDetection -> ContactRequest -> ContactBoundary -> FormationField`, пока без Avian, но уже используется production simulation и test harness; `ContactFront` строится из реальной `Formation` geometry. Есть CI для `fmt`, `clippy` и `cargo test`. Пользовательская оценка первой Layer 1 версии — приемлемо; теперь калибруем поведение формаций и готовим Contact Zone перед подключением физдвижка. Это ещё не игра и ещё не доказательство полной HYP-001.

## Ядро симуляции

**FMP (Formation Material Physics)** — формация как физический материал; мораль / организованность / усталость = механические свойства (yield, stiffness, деформация). Три слоя активации:

| Слой | Роль | Стоимость |
|------|------|-----------|
| 1 — Formation Field | Поле, волны давления, паника | O(formations) |
| 2 — Contact Zone | Полная физика переднего ряда | O(ширина контакта) |
| 3 — Individual | Герой, удары, слом, вне строя | O(события) |

Удары — **GIC** (геометрический импульс из shapecast), не «урон по таймеру анимации».

## Неприкосновенные принципы (кратко)

- Поведение важнее голых чисел; формации — **режимы поведения**, не баффы.
- Физика — источник истины; анимация — отображение.
- Управляемость и причинность для игрока важнее «абсолютной» симуляции.
- Не MOBA про скиллы, не RTS-мясорубка, не микроконтроль каждого бойца.

## Технический вектор (зафиксирован в заметках)

- **Bevy 0.18**, не гнаться за апдейтами во время лаборатории.
- **Avian** для слоёв 2–3; слой 1 — кастомная система.
- **3D, вид сверху** сразу (лаборатория как 2D, стек без миграции avian2d→3d).
- **leafwing-input-manager**, **bevy_egui** (debug UI лаборатории).
- **Lightyear** — мультиплеер, не сейчас.

## MVP / лаборатория

Доказать: ценность боя, формаций, морали, тактики, коопа — через **работающую симуляцию FMP**, а не объём контента. Следующий шаг — проверить читаемость анизотропии в сценариях «клин» и «фланг», добавить контрастные архетипы формаций (например, Phalanx / Crowd), затем подключать contact zone.
